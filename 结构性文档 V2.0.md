## 1. 产品一句话

**蓝图编排者 Blue-Map Orchestrator** 是一个基于大模型与系统能力的个人旅行导演 Agent。

它能从语音、文本、PDF、截图中理解用户意图，结合 **出行偏好结构化问卷** 与 **高德 POI/路线真实数据**，生成可编辑的弹性时空拓扑行程；支持 **多平台候选对比 → 用户选店定节点 → 真实费用汇总**；并通过 deeplink 跳转携程、美团、大众点评、小红书、高德等应用完成跨端执行（支付链路当前为模拟授权）。

## 2. 核心用户场景

### 场景 A：经典出差规划

用户输入：

> 「下周五去北京出差三天，住西单附近，想吃正宗烤鸭，周六下午想去故宫。」

系统自动完成：

1. 解析目的地、相对日期（明天/下周五等）、五要素与系统上下文。
2. 读取用户 **出行偏好**（同行、风格、节奏、住宿、时间安排）。
3. 生成 3 套候选方案（最快 / 均衡 / 低预算），**总价由价格引擎实时计算**，非写死数字。
4. 展示时空拓扑看板 + 行程时间轴。
5. 用户可编辑、拖动、删改、重排节点；Agent **智能联动** 后续时间与描述。

### 场景 B：云南菌子火锅 + 选店定节点

用户输入：

> 「去云南吃菌子火锅，帮我对比哪家更好吃。」

系统执行：

1. 生成基础行程后，用户点击 **「推荐美食」** 或餐饮节点上的 **「选」**。
2. 后端调用 **高德 POI 搜索** + LLM 排序，返回候选餐厅列表。
3. 每条候选展示：评分、预估人均、距上一节点距离/时长、**多平台参考分**（高德/大众点评/美团/小红书/携程）。
4. 用户可 deeplink 跳转各 App 查看详情，选定后 **「选这家并定节点」** 写入行程。
5. **价格引擎** 重算交通（高德驾车路线 + 打车费）、餐饮、住宿合计。

## 4. 阶段一：意图爆发期（已实现细节）

### 4.1 输入方式

| 输入类型 | 前端实现 | 后端处理 |
| --- | --- | --- |
| 长文本 | `IntentInputPanel` TextInput | `/intent/analyze` → DashScope qwen-plus |
| 语音 | expo-audio 录音 + 设备/Web/云端 ASR 三级降级 | `/voice/transcribe-base64`（百度 ASR / DashScope） |
| 文件 | expo-document-picker | `/uploads` → RAG 分片入库 |
| 结构化字段 | 出发地/目的地/日期 | 合并进 `TravelRequestBundle.structured` |
| **出行偏好** | `TravelPreferencesModal` 全屏问卷 | 序列化进 `preferences` 与 `tags` |

### 4.2 出行偏好（参照竞品逻辑，V2 新增）

入口：意图输入页 **「出行偏好 → 编辑 ›」**

| 分类 | 选择方式 | 选项 |
| --- | --- | --- |
| 同行伙伴 | 单选 | 独自出行 / 家庭出行 / 情侣出行 / 朋友出行 / 老人同行 |
| 风格偏好 | 多选 | 文化体验 / 经典必去 / 自然风光 / 城市景观 / 历史古迹 |
| 行程节奏 | 单选 | 紧凑 / 适中 / 宽松 |
| 住宿偏好 | 单选 | 舒适型 / 高档型 / 豪华型 |
| 时间安排 | 多选 | 偏早出 / 偏晚归 |
| 其他 | 文本 | 最多 1000 字 |

**数据流：**

```text
TravelPreferencesModal
  → travelPreferences 状态 (TravelDirectorScreen)
  → serializeTravelPreferences() → structured.preferences
  → buildTravelRequest() → preferencesToApiList() + preferencesToTags()
  → 后端 TravelIntent.preferences / PlanComparisonService
```

**默认偏好：** 独自出行 · 适中 · 舒适型

### 4.3 五要素抽取

| 要素 | 字段来源 | 展示位置 |
| --- | --- | --- |
| Action | preferences + 规则推断 | `IntentAnalysisPanel` |
| Location | origin / destination / must_visit | 同上 |
| Time | start_date / end_date + **相对日期解析** | 同上 + 节点日期标签 |
| Constraint | constraints | context 洞察 |
| Preference | 出行偏好 + 文本解析 | 同上 |

**相对日期（V2）：** 今天 / 明天 / 后天 / 大后天 / 下周五 / 下周X —— 前后端均已支持（`date_resolve.py` + `parseTravelInput.ts`）。

### 4.4 语音链路（V2 稳定版）

```text
优先级：设备原生识别 (expo-speech-recognition，Expo Go 自动跳过)
      → WebView Web Speech（Web 端）
      → 云端录音 ASR（expo-audio m4a → base64 → 百度/DashScope）
```

配置：`SPEECH_PROVIDER=baidu|dashscope|auto`，百度需 `BAIDU_ASR_*` + `chardet` 依赖。

## 5. 阶段二：视觉转译期（已实现细节）

### 5.1 方案比对

- API：`POST /plans/compare`
- 服务：`PlanComparisonService` → LangGraph 生成基础行程 → **price_engine 真实报价** → 3 套策略变体
- 策略：最快抵达（×1.08）/ 均衡舒适（×1.0）/ 低预算（×0.88）
- **不再写死 1280/1580/980**，总价 = 交通 + 餐饮 + 住宿 + 其他

### 5.2 时空拓扑看板

| 组件 | 功能 |
| --- | --- |
| `MapTopologyBoard` | 高德 WebView 地图，节点标注，拖动触发 smart-update |
| `ItineraryTimeline` | 时间轴：编辑 / ↑↓ 排序 / 删除 / **「选」POI** |
| `TopologySummary` | 硬锚点 / 半硬锚点 / 软任务 / 风险计数 |
| `NodeEditModal` | 编辑标题、时间、地点 |

### 5.3 节点分级（代码字段）

| 类型 | `node_type` / category | 可编辑 | 示例 |
| --- | --- | --- | --- |
| 硬锚点 | hard_anchor / transport, meeting, hotel | 部分 | 航班、会议、酒店入住 |
| 半硬锚点 | semi_anchor / food, sight | 是 | 已选餐厅、景点 |
| 软任务 | soft_task / free | 是 | 弹性活动 |
| 提醒 | alert | — | 守护重排插入的缓冲 |

## 7. 阶段四：跨端执行期（已实现细节）

### 7.1 模拟订票流程

```text
prepare → authorize-payment (mock) → execute → sync/system
```

`ExecutionCenter` 填充步骤：信息解析 → 平台匹配 → 参数填表 → 等待授权 → 结果回调。

### 7.2 系统同步

`SystemSyncService` 生成：日历 / 提醒 / Widget / 备忘录 / 地图 deeplink。

### 7.3 多平台 POI 与 deeplink（V2 核心）

**文件：** `apps/api/app/integrations/deeplinks.py`

| 平台 | 用途 | 实现方式 |
| --- | --- | --- |
| 高德 | POI 搜索、驾车路线、地图 marker | **REST API（真实，需 AMAP_API_KEY）** |
| 大众点评 | 餐厅搜索跳转 | deeplink |
| 美团 | 商户搜索跳转 | deeplink |
| 小红书 | 攻略/口碑搜索 | HTTPS 搜索页 |
| 携程 | 酒店/度假搜索 | HTTPS / deeplink |

> **诚实说明：** 除高德外，小红书/携程/大众点评/美团 **无公开免费消费级 API**；当前为 **deeplink + 基于高德评分的参考分**，非实时爬取各平台数据。

### 7.4 POI 推荐 → 定节点流程

```text
POST /recommendations/search
  ├─ amap_service.search_poi(city, keyword, category)
  ├─ amap_service.route_estimate(上一节点 → 候选点)
  ├─ attach_platform_scores() 多平台参考分
  └─ LLM 排序 + 推荐理由

用户选定 → POST /nodes/confirm-poi
  ├─ recommendation_service.confirm_selection()
  ├─ 写入 ItineraryItem（estimated_cost, booking_source, booking_deeplink）
  └─ price_engine.quote_itinerary() 返回最新报价
```

**移动端：** `OptionPickerModal` 展示候选列表、平台分、跳转按钮、「选这家并定节点」。

## 8. 真实费用计算（V2 新增）

### 8.1 价格引擎

**文件：** `apps/api/app/services/price_engine.py`

| 费用项 | 来源 |
| --- | --- |
| 餐饮 | 节点 `estimated_cost` 或默认 ¥100/人 |
| 住宿 | 节点 `estimated_cost` 或默认 ¥380/晚 |
| 交通 | 相邻节点间 **高德驾车路线** taxi_cost + tolls |
| 合计 | transport + food + hotel + other |

**API：** `GET /itineraries/{id}/price-quote` → `ItineraryPriceQuote`

**移动端：** 拓扑看板上方「真实费用估算」卡片，选店后自动刷新。

### 8.2 与方案比价的关联

`PlanComparisonService._option()` 基于 `price_engine.quote_itinerary(base)` 生成三档方案，推荐文案含分项金额与数据来源（amap / estimate）。

## 15. 答辩核心卖点（更新版）

1. **不是攻略生成器**，而是可编辑、可联动的 **行程导演 Agent**。
2. **不是写死报价**，而是高德路线 + 节点成本 **实时汇总**。
3. **不是单平台推荐**，而是 **多平台候选对比 → 用户确认 → 定节点** 的决策闭环。
4. **不是一次性规划**，而是 smart-update + 守护重排的 **持续微调**。
5. **不是纯文本偏好**，而是 **结构化出行偏好问卷** 驱动方案生成。
6. **诚实标注** 哪些数据来自高德真实 API、哪些为 deeplink/估算，体现工程可信度。
